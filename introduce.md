# 📄 产品需求文档 (PRD): SilentStats Web

**目标：** 记录用户手机处于"无声状态"的总时长（即用户远离抖音、B站等干扰应用的时间）。

**核心理念：** 放弃"实时监控"，采用"端点追溯"和"焦点争夺"逻辑。

---

## 1. 核心技术逻辑

### 1.1 音频焦点计时

**原理：** 利用浏览器音频焦点机制实现计时。

- 网页播放循环 白噪音 音频（url），只要音频在播放，JavaScript 就持续运行并计时。
- 用户锁屏或切后台，音频继续播放，计时不中断。
- 当用户打开抖音、B站等有声 App 时，系统会抢占音频焦点，触发 `onpause` 事件，立即停止计时。

### 1.2 干扰应用探测 (Audio Focus Conflict)

**痛点：** 网页无法知道用户在后台开了什么 App。

**对策：** 利用浏览器音频焦点机制。

- 网页开始时播放一段循环音频（url），占住音频焦点。
- 只要用户开启抖音、B站、网易云音乐等有声 App，系统会强制抢占焦点并暂停网页音频。
- 网页监听 `onpause` 事件，记录打断开始时间，状态显示"检测到干扰"。
- 打断后**不主动抢回**，等用户回到页面后手动点开始继续。

---

## 2. 功能清单

### 2.1 任务执行逻辑

**【开始挑战】**
- 用户点击按钮触发音频播放（浏览器要求用户交互才能播放）。
- 记录 `sessionStartTime` 时间戳，开始计时。

**【音频被打断】**
- 音频触发 `onpause` 事件（用户打开抖音、B站等 App 抢占音频焦点）。
- **立即停止计时**，记录本次专注时长。
- 状态显示"检测到干扰"，发送系统通知（需 HTTPS + 用户授权）。

**【继续挑战】**
- 用户手动点击"开始挑战"按钮，音频重新播放，开始新的计时。

**【暂停专注】**
- 用户手动点击"暂停专注"按钮，停止音频，暂停计时。
- 记录本次专注时长到时间轴。

### 2.2 今日专注计数

- 顶部实时显示今日累计纯净专注时长（精确到秒）。
- 只有音频播放且未被打断时才累加。
- 以日期为 key 存入 localStorage，每天自动清零。

### 2.3 今日时间轴

- 24 小时横条，绿色块表示每次专注时段。
- 正在进行的时段闪烁显示。
- 条目列表显示每段的开始→结束时间和时长。
- 每次只有一个"进行中"状态，新开始前自动关闭未结束的记录。
- 数据存入 localStorage，刷新不丢失，一直保存。

### 2.4 系统通知

- 被打断时发送通知"检测到干扰"。
- 需要 HTTPS 环境 + 用户授权（iOS Safari 不支持）。

---

## 3. 技术栈

- **前端：** 原生 HTML / CSS / JavaScript
- **样式：** Tailwind CSS（CDN 引入）
- **音频：** 本地 url，循环播放
- **存储：** localStorage（无数据库，无后端）
- **部署：** Vercel / GitHub Pages（静态托管，需 HTTPS 以支持通知）

---

## 4. 判定逻辑

| 音频状态 | 系统行为 |
|---|---|
| 音频播放 | 正常计时，计入纯净时间 |
| 音频暂停 | **立即停止计时**，状态显示"检测到干扰"，需手动重新开始 |
| 音频打断 | 同"音频暂停"，触发 `onpause` 事件 |
